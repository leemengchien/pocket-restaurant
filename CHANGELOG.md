# 更新說明

口袋餐廳（多使用者版）功能演進紀錄。最新在最上面。

## 2026-07-03

### 匯入強化
- **修正筆記匯入**：Google Takeout 的 CSV 用「筆記」欄，舊版只認「附註」而漏掉，現已正確帶入，且搜尋找得到筆記內容。
- **匯入時自動抓 Google 資料**：每筆用店名查 Google，一次帶入星等、評論數、地址、電話、營業時間、用戶回報價位（自動對應系統的 $–$$$$）、Google 分類。
- **料理分類自動標**：用 Google 判斷的類型（中文化，如「拉麵・餐廳・宵夜」）自動貼多個標籤，可自行增刪。
- **國家 / 縣市 / 地區**：依地址自動判斷並分三層存放；日本英文地址（如 Osaka）也會轉成中文（大阪）。

### 餐廳卡片
- 顯示星等、地址、電話、可展開的營業時間、國家・縣市・地區、料理標籤。
- 按鈕：🌐 看評論（連 Google 商家頁）、📋 看菜單（Google 搜尋）、📝 查部落格（Google 搜尋店名＋食記）。

### 編輯表單
- 匯入的地址會自動帶入「位置」欄位。
- 可編輯「料理標籤／關鍵字」（多個、逗號分隔）。
- 可編輯「國家 / 縣市 / 地區」，自動判斷不出來時能自己選或填。

### 搜尋與篩選
- 附近清單上方新增 **國家 → 城市 → 地區** 三層連動篩選。
- 篩選器內含「⚠️ 未分類」選項，一鍵列出分類空白的餐廳方便補齊。
- 搜尋頁新增 **地區（國家／城市）**、**狀態**、以及既有的價位篩選；選了地區就不受距離限制。
- 搜尋頁新增「**要去別的城市／地點？**」：輸入地點就能搜當地 Google 高評價餐廳（沒記錄過也能找）。
- 「改定點」與「地圖選點」的地圖視窗可用文字**搜尋地點／地址／地標**，圖釘自動移過去。
- 搜尋範圍涵蓋：店名、料理類型、料理標籤、推薦菜、備註、地址、縣市、地區。

### 工具
- 設定頁「🔧 重新分類國家／縣市／地區」：用已存地址重新解析分類，**不呼叫 Google、不花額度**，可修正早期匯入分類空白或不對的資料。

### 資料庫欄位（Supabase）
陸續新增：`rating`、`rating_count`、`place_id`、`address`、`phone`、`hours`、`tags`、`google_uri`、`country`、`city`、`district`。
對應 migration：`002`～`005`（在 Supabase SQL Editor 依序執行）。

## 初始版本

- Google 登入、雲端同步（Supabase + RLS，每人只看自己的資料）。
- 附近清單、口袋名單搜尋＋Google 高評價搜尋、50m 停車查詢。
- 記錄／編輯、自訂選項（狀態／價位／停車）、JSON 備份、Google Maps CSV 匯入。
- Places API 走後端代理，API key 只存伺服器、未登入無法呼叫。
